Prep for open item #4 (dead-output prune). Enumerates all **448** files in
`figs/` and classifies each relative to `output_manifest.csv`.

## Key conclusion: a static prune is unsafe

The 413 untracked files are **not** dead. Nearly all are produced by scripts via
**dynamic filenames** the static scan can't match literally:

- The s3 engine (`s3_predicValuesPGLS_MERGED_variant.R`) writes per-region jpgs in
  a loop: `paste0("phylo_gls_", safe_region, ".jpg")`, and general plots via a
  `save_plot(name, ...)` helper. `s3_run_and_compare_configs.R` re-runs the engine
  once per configuration (`all`, `cortical`, `telencephalon`, `merged_all`,
  `merged_all_anthro`, `merged_histological_anthro`, `stephan_anthro`, plus
  `s3v2`), so the same plot names are written into ~8 config subdirs.
- s1a stereology pies and several s1b E:I / astrocyte variants are likewise
  dynamic or extra renders.

So "untracked" here means "generated by the config sweep / not a named deck
figure," not "orphaned." The only reliable way to find genuinely dead outputs is
the **clean-room test** in open item #4: move `figs/` aside, run
`Rscript scripts/run_all.R`, and list whatever does **not** reappear. That
requires R, which is not installed in the Cowork sandbox.

## How to use the CSV

`deck_relevance` column flags the likely prune targets once the clean-room run
confirms reproducibility:

- `manifest_output` → keep (embedded in deck).
- s3 rows in `figs/s3/all` → `maybe (core config)` — the config the deck slides
  37–38 draw from; keep the core, review the rest.
- s3 rows in other config dirs → `no (diagnostic config sweep)` — candidate for
  archiving if the config isn't needed for the paper, **after** confirming
  `run_all.R` regenerates them on demand.
- non-s3 untracked → `variant/extra render` — review case by case.

Do not delete based on this file alone.
